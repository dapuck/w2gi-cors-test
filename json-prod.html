Hello CORS!

To make a CORS compatible request to W2GI's systems you will first need to obtain a CORS enabled Appkey from W2GI (contact your account rep for more details).

To make the request, it is best to use a library that supports AJAX requests using CORS (like jQuery). Please note that not all browsers support CORS and some have only partial support, see http://caniuse.com/#feat=cors for more information.

In this example we will use jQuery to send a prepared AJAX request to api.where2stageit.com. All of the code shown here can also be found in the source for this page. As you can see below, this code adds an on click handler to the element “#doit” that will call the function “doRequest”. The function “doRequest” will use jQuery's “ajax” function to make the AJAX request to api.where2stageit.com, passing in the prepared JSON. 

      (function($){
        var url = "http://api.slippymap.com/rest/getlist";
        var appkey = "A3DEE464-3DC0-11E4-AFDD-17AF746343CC";
        var req = {
            "request": {
                "appkey": appkey,
                "formdata": {
                    "objectname" : "Country",
                    "order": "name"
                }
            }
        };
        function doRequest() {
          function success(data, status, xhr) {
            $("#response").text(JSON.stringify(data, null, 4));
          }

          function error(xhr, status, err) {
            alert(err);
          }
          if(window.XDomainRequest) {
            //console.log("Old IE CORS");
            var xdr = new XDomainRequest();
            xdr.open("post", url);
            xdr.onprogress = function () { };
            xdr.ontimeout = function () { };
            xdr.onerror = error;
            xdr.onload = function () {
              $("#response").text(xdr.responseText);
            };
            setTimeout(function () {xdr.send(JSON.stringify(req));}, 0);
            return;
          }
          $.ajax(url, {
            method : "POST",
            data : JSON.stringify(req),
            contentType : "application/json",
            success : success,
            error : error
          });
        }

        $(function(){
          $("#doit").on("click", doRequest);
        });
      })(jQuery);

If the Appkey has been configured correctly for the domain, the response should be displayed in the element “#response”. Otherwise the error will be displayed as an alert.
